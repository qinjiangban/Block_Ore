## 1. 页面设计目标
页面设计围绕“单手可玩、两秒反馈、链上可信、结果有惊喜”四个目标展开，确保 Block Ore 在移动端具备极强的可进入性和持续点击欲望。
- 所有主路径控制在三步以内：打开首页、开始挖矿、查看结果。
- 所有关键反馈必须即时可见，包括点击、动画、等待、Reveal、积分变化和奖励高光。

## 2. 视觉系统
### 2.1 氛围定义
- 主题情绪：深空矿洞、数字探矿、熔岩金属。
- 背景结构：深色渐变底叠加低透明噪点、扫描线和矿脉流光。
- 层级方式：卡片悬浮于主背景之上，使用多层阴影与边缘高光制造“矿石舱”质感。

### 2.2 色彩与材质
| 用途 | 颜色 | 说明 |
|------|------|------|
| 页面底色 | `#090B12` | 近黑蓝底，承载深空氛围 |
| 卡片底色 | `rgba(23,27,39,0.78)` | 半透明矿岩卡片 |
| 主强调色 | `#4DA8FF` | 用于按钮、可交互状态、链上扫描线 |
| 次强调色 | `#F9B84E` | 用于奖励、矿镐价格、进度完成态 |
| 危险/失败色 | `#FF6B7A` | 用于链上失败与错误提示 |
| 稀有度色 | `#B38CFF` / `#7AE7FF` / `#FFF2A6` | 用于 Diamond / Genesis / 高光边框 |

### 2.3 字体层级
- 页面标题：展示型标题字体，字重 600-700，用于品牌与结果高潮。
- 常规正文：易读无衬线字体，字重 400-500。
- 仪表盘数字：等宽数字风格，强化积分、次数和区块编号的科技感。
- 字号体系：12 / 14 / 16 / 20 / 28 五档为主，避免过多层级。

## 4. 动效设计
### 4.1 首页动效
- 顶部 Logo 轻微呼吸发光。
- 数据卡片在页面载入时分段上浮并伴随数字滚动。
- 主按钮悬停或按压时出现内发光与轻微缩放。

### 4.2 挖矿页动效
- 矿石空闲时以 3-5 秒周期呼吸缩放并带粒子漂浮。
- 点击矿石后立即出现冲击波、裂纹扩散和短暂屏幕震动。
- Reveal 结果按稀有度控制光效强弱，Genesis 额外叠加全屏闪光与星屑。

### 4.3 成就与排行榜动效
- 成就完成时展示徽章点亮与进度条冲刺动画。
- 排行榜切换采用短距离滑动转场，避免页面整体重绘感过重。

## 5. 组件拆分建议
- `AppShell`：统一移动端容器、背景和底部安全区。
- `PlayerStatsCard`：积分、剩余次数、累计挖矿次数卡片。
- `MineCrystal`：矿石主视觉与点击动效封装。
- `RevealTimeline`：mine 到 reveal 的流程进度条。
- `OreResultCard`：结果与奖励展示卡片。
- `OreInventoryGrid`：矿石库存网格。
- `AchievementList`：成就列表。
- `LeaderboardTabs`：排行榜切换与榜单列表。
- `PickaxeShopSheet`：矿镐商店弹层。

## 6. 交互文案基调
- 文案风格短促直接，避免冗长说明。
- 状态提示以行动导向为主，例如“等待 2 个区块”“立即 Reveal”“今日次数已满”。
- 奖励文案需要更强情绪，例如“发现钻石矿”“创世矿已锁定进背包”。
